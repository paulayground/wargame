# Problem Overview

홈 디렉토리에 `setuid`가 설정된 `suconnect`파일이 있으며,
포트를 열어 `suconnect`를 통하여 접속하고, 현재 비밀번호 전송한다면 다음 비밀번호를 얻을 수 있음.

# Analysis

`suconnect` 실행 시 포트번호와 같이 사용하라는 힌트를 얻었음.
`suconnect`에 `setuid`가 설정되어 있어 실행 시 다음 비밀번호가 저장된 bandit21권한의 파일을 읽어 응답할 것으로 예상함.

> 서버 세션에서 임의의 포트를 열고 별개의 세션에서 생성한 클라이언트에서 내가 만든 서버에 접속해 서버에서 메시지를 전달하면 suconnect를 통해 다음 비밀번호가 전달되는 과정을 확인하라는 것과 프로그램이 서버에서 데이터를 가져오는 구조라면 공격자 서버를 만들어 해당 프로그램을 연결해 입력을 제어할 수 있는 부분을 확인하라는 것으로 보임.

# Exploitation Strategy

안쓰는 포트를 열어 대기하고 클라이언트 세션을 하나 더 열어 `suconnect`를 통해 접속하고 현재 비밀번호를 보내면 다음 비밀번호를 얻을 수 있을 것이라 판단함.

# Execution

```bash
# 서버 측 포트 리슨
nc -lvp 40000

# 클라이언트 측 포트 접속
./suconnect 40000

# 서버 측에서 현재 비밀번호 전송
```
