# 배운 점

`more`에서 `v`를 통해 vi 편집기로 넘어갈 수 있으며, `more`도 `vim`처럼 다양한 명령키워드가 있는 것을 알게됐으며 외부 명령(`!COMMAND`)도 실행가능한 것을 확인했다.
vi 편집기에서 명령모드(`:`)를 통해 외부 명령실행(`:!COMMAND`)이나 외부 파일을 편집(`:e FILE`)할 수 있다는 것을 알게됐다.

# 풀이 과정 기록

bandit풀면서 가장 오래걸린 것 같은데, 어떻게든 힌트 없이 풀어보려고 하다 너무 오래걸렸다.
나중되서야 알았지만 bandit26에 접속할 때도 bandit26의 텍스트로 이루어진 그림이 출력되는데, 그 당시에는 로그인 프롬프트 마냥 자연스레 무시하며 지나갔다. 후에 bandit25의 로그인 화면이랑 비교하니까 차이점을 알았다.
이 때 잘봤으면 `more`에 대한 터미널 크기 때문에 접속이 종료된다는 사실을 빨리 깨닫지 않았을까 아쉽기도 했다.
로컬에서도 비슷한 파일을 만들어 테스트하는데도 알아차리지 못한거는 아쉬움이 남는다.
검색을 통해 로그인 쉘이 `more` 상태일 때의 문제들을 찾아보면서 터미널창의 크기에 대한 힌트를 얻고 more -> vi, vi에서 외부명령, 외부파일 편집 등을 찾아보며 문제를 해결했다.

# 심화 학습 (Deep Dive)

more 진행 중에서 v를 눌러 vi편집기로 넘어 간 후 vi 명령모드에 외부 파일 수정을 위한 `:e /etc/bandit_pass/bandit26`를 통해 해당 파일을 요청하면 다음 비밀번호를 확인할 수 있다.

`vi`이나 `more`에서 `!`를 이용해 명령을 실행할 때, 편집기는 사용자의 `$SHELL`을 호출해서 명령어를 실행하는데, `$SHELL`이 `/bin/bash`가 아닌 `/usr/bin/showtext`로 설정되어 있어서 `more`이나 `vi`에서 외부 명령을 실행해도 정상적인 결과를 얻을 수 없었다.
