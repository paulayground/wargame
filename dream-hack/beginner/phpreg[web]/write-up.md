1. php step 1코드에 닉네임과 비밀번호를 제출하여 step2로 넘어가는 코드를 확인하였다.
2. step2의 닉네임 정규표현식이 `/nyang/i` 이 포함되면 공백처리되게 되는 구성이 되어있고 통과 닉네임으로 `dnyang0310`이 지정되어있어 정규표현식을 피하면서 닉네임을 맞출 수 있게 `dnyannyangg0310`와 같이 정규표현식에 걸려 공백으로 대체 되어도 닉네임이 완성될 수 있게 구성하였다.
3. 비밀번호의 정규표현식의 경우 `/\d*\@\d{2,3}(31)+[^0-8\"]\!/`와 같이 구성 되어있어 정규표현식에 맞는 문자를 입력하면 `d4y0r50ng`으로 변경되는 것을 확인하였다. 통과 비밀번호가 `d4y0r50ng+1+13` 로 이루어져 있어 정규표현식과 일치하는 `0@00319!+1+13`와 같은 비밀번호를 입력하였다.
4. 닉네임과 비밀번호 입력 후 나온 cmd input 창에서 문제에 제시된 `../dream/flag.txt` 를 확인하고 접근하였으나 `/flag/i` flag문구를 정규표현식으로 막아놓은 것을 `?` 와일드카드로 `../dream/fl?g.txt` 와 같이 대체하여 `cat ../dream/fl?g.txt`를 통해 flag 획득하였다.
